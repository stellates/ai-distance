# article-update-spec.md

## 本書について

本書は、以下の作品の記事更新作業を行う手順書である。

### 作品名

AIとのちょうどいい距離感（AI-DISTANCE）

### 作品についての概要

「AIとのちょうどいい距離感」は、生成AIとの付き合い方や活用時の注意点を、漫画や短い記事を通じて伝えるコンテンツである。

生成AIを過信することや必要以上に恐れることを避け、利用者が適切な距離感を持てるような情報発信を目的としている。

各記事は原則として1話1テーマで構成され、生成AIに関する身近な事例や気付き、学びなどを扱う。

---

## 作業実施環境要件

本手順は以下の環境が整っていることを前提とする。

初回実施時は、必要なソフトウェアの導入、リポジトリの配置、および権限設定を事前に完了しておくこと。

### 作業端末

- パソコンを利用すること
- インターネットへ接続できること

### 利用サービス

- ChatGPT を利用可能であること
- ChatGPT はブラウザ版またはデスクトップアプリ版を利用できること
- Plus または Business 以上の有償プランを推奨

### 必要ソフトウェア

- Visual Studio Code（VSCode）
- Git
- Google Chrome

### リポジトリ環境

- ai-distance リポジトリがローカルに配置されていること
- リポジトリを編集可能な状態であること

### 権限

- GitHub リポジトリへ push 可能であること

## 作業工程ワークフロー

| 工程番号 | 工程名 | 作業概要 |
|----------|--------|----------|
| 00 | 事前準備 | local gitの最新化、AIの作業向けウォームアップを行う。 |
| 01 | 企画 | 記事テーマを決定する。以降の構成設計や画像設計へ繋げるため、大まかなテーマや方向性を整理する。 |
| 02 | 構成設計 | 漫画全体のストーリー、オチ、学び、読後感を設計する。画像化する前に構成を固める。 |
| 03 | 記事インベントリ追加 | 記事番号、タイトル、テーマ、ステータスなどを記事管理情報へ登録し、制作対象として管理を開始する。 |
| 04 | 画像設計 | コマ構成、セリフ、キャラクター配置、演出、画像サイズなどを整理し、シリーズの作風を損なわない共通指定も含めて画像生成の設計を行う。 |
| 05 | 画像作成 | 画像生成を実施し、人間による品質確認と必要な修正を行う。 |
| 06 | 記事作成 | 記事HTMLを作成し、タイトル、本文、画像、メモ、「今回の距離感」などの記事内容を作成する。 |
| 07 | 一覧更新 | トップページや記事一覧へ新しい記事へのリンクや記事カードを追加する。 |
| 08 | 確認 | 作成した画像を所定のフォルダへ配置し、ローカル環境で表示確認を行う。レイアウト、リンク、画像表示、誤字脱字などを確認する。 |
| 09 | 公開 | Gitへコミットし、GitHubへpushして公開する。 |
| 10 | 公開確認 | GitHub Pages反映後の表示確認を行い、公開内容に問題がないことを確認する。 |

### 01_企画

#### 入力情報の準備

- 特になし

#### インストラクション

1. 1話で扱う生成AIに関するテーマを決定する。
2. テーマは1話につき1つに絞る。

テーマ例：

- 存在しない論文をそれっぽく引用する
- 数字を自信満々に言う
- 法律相談にそれっぽく答える
- 最新情報なのに確認せず答える
- 人間がAIを過信してしまう
- AI導入を怖がりすぎる会社

#### 出力情報

- 記事テーマ

#### 工程完了判断のチェック項目

- 文字量が多すぎないか
- テーマが伝わるか
- 既存記事と雰囲気がズレていないか
