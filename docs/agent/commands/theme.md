# /theme

1記事1テーマで、記事と画像の設計を固める。

## 入力

```text
/theme NNN
```

## 目的

「AIとのちょうどいい距離感」らしい新記事テーマを作るための考え方です。

テーマを出力することが目的ではありません。

既存記事と重複せず、

- 共感できる
- 4コマ化できる
- 一言で学びがある

テーマを作ることを目的とします。

## シリーズ基本構造

このシリーズは以下の3段構造で考えます。

1. AIの特徴・挙動
2. 人間が困るポイント
3. ちょうどいい付き合い方

例:

AIが略語を使う
↓
意味が分からない
↓
「略語なしで説明して」と頼む

AIが説明しすぎる
↓
知りたい場所が見えない
↓
「図で説明して」と頼む

この3つが揃わないテーマは採用しません。

## テーマ発想の優先順位

以下の順でテーマ候補を探します。

1. AI利用者が実際によく遭遇する小さな違和感
2. AIの失敗ではなく、人間との認識ズレ
3. 一言で改善できること
4. AI初心者でも共感できること

優先度が低いもの:

- AI業界ニュース
- モデル比較
- 技術解説
- APIやプログラミング前提
- AI賛美
- AI批判

## テーマ作成手順

1. docs/skills/post_inventory.md を読む
2. 既存記事の傾向を把握する
3. まだ扱っていないAIあるあるを探す
4. 類似記事を特定する
5. 重複回避ポイントを作る
6. タイトル案を作る
7. 4コマ化できるか確認する
8. 今回の距離感を1文で表現する

## 良いテーマ判定

以下を満たすほど良いテーマです。

- 「それある！」と言える
- AI初心者でも理解できる
- 4コマ漫画にしやすい
- 今回の距離感を1文で言える
- 説教にならない
- AIを過信させない
- AIを怖がらせすぎない

## NGテーマ

以下は避けます。

- AIが間違えた
- AIが嘘をついた
- AIが賢い
- AIがすごい

理由:

テーマが広すぎます。

必ず、

- どんな場面で
- 何が起きて
- どう距離感を取るか

まで具体化します。


## 重複チェック

テーマ提案時は必ず以下を出力します。

- 似ている既存記事
- 重複しない理由
- 今回だけの新しさ

同じ教訓を別タイトルで繰り返さないことを重視します。


## 出力に含めるもの

- テーマ
- タイトル案
- サブタイトル案
- AIの挙動
- 人間が困るポイント
- 今回の距離感
- 4コマ構成案
- セリフ案
- 似ている既存記事
- 重複しない理由
- 今回だけの新しさ
