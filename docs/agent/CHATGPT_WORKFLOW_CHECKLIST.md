# CHATGPT_WORKFLOW_CHECKLIST.md

このファイルは、ChatGPT が「AIとのちょうどいい距離感」の新記事追加を人間と一緒に進めるときの工程管理チェックリストです。

目的は、作業の現在地を人間にも ChatGPT にも見えるようにし、工程飛ばし・重複テーマ・HTML先走り・画像先行による手戻りを防ぐことです。

---

## 基本方針

ChatGPT は、新記事作成支援の主要な返信で、現在の進行状況を短いチェックリストとして表示します。

表示は以下を使います。

- `[x]` 完了
- `▶` 現在作業中
- `[ ]` 未着手

ただし、ユーザーが短い回答を希望している場合は、詳細説明を省き、チェックリストも必要最小限にします。

---

## 進行状況チェックリスト

```md
進行状況:
- [ ] 0. AGENTS.md確認

- [ ] 4. 新記事テーマ決定

- [ ] 6. posts/NNN.html草案作成
- [ ] 7. index.html追加カード草案作成
- [ ] 8. POST_INVENTORY.md追記草案作成
- [ ] 9. 画像生成設計

- [ ] 11. 最終HTML調整
- [ ] 12. ローカル確認
- [ ] 13. commit / push / 公開確認
```

## 各工程の完了条件

### 0. AGENTS.md確認

完了条件:

- `AGENTS.md` を読んでいる
- 参照すべきガイドファイルを把握している
- 作業者が ChatGPT / 人間補助モードか、Codex / 自動作業モードかを判断できる状態になっている

---

### 1. 作業モード確認

完了条件:

- 原則として ChatGPT / 人間補助モードで進めるかを確認している
- ChatGPT が repository を直接編集しない前提か、人間の明示依頼により限定的に編集するかを確認している
- commit / push / PR / merge を行うか行わないかが明確になっている

通常の人間補助モードでは、ChatGPT は差分・本文・手順を提示し、人間がローカル clone 上で反映します。

---

### 2. 既存記事棚卸し確認

完了条件:

- `index.html` で最新記事番号を確認している
- `posts/` の既存記事、または `docs/agent/POST_INVENTORY.md` を確認している
- 既存記事の番号、タイトル、テーマ、今回の距離感、タグの傾向を把握している
- 最新3記事だけでなく、過去記事との重複確認に必要な情報を確認している

`POST_INVENTORY.md` が存在しない、または古い場合は、推測で進めず、作成・更新を提案します。

---

### 3. テーマ重複チェック

完了条件:

- 新テーマ案に近い既存記事を確認している
- 似ている既存記事がある場合、その記事番号とタイトルを示している
- 重複しない理由、または避けるべき方向を短く説明している
- 「同じ教訓を別タイトルで繰り返すだけ」になっていないことを確認している

テーマ提案時は、必ず以下を短く示します。

- 似ている既存記事
- 重複しない理由
- 今回だけの新しさ

---

### 4. 新記事テーマ決定

完了条件:

- 1記事1テーマに絞れている
- 日常の小さな「AIあるある」として説明できる
- 生成AIを過信させず、怖がらせすぎない方向になっている
- 説教臭くなく、軽いぼやき調にできる見込みがある
- 人間がテーマを了承している

この段階では、まだHTML全文や画像生成プロンプトを確定しません。

---

### 5. 記事構成設計

完了条件:

- 記事番号が決まっている
- タイトルが決まっている
- サブタイトルが決まっている
- 1記事1テーマの説明が決まっている
- メモ本文の流れが決まっている
- 今回の距離感が1文で決まっている
- タグが決まっている
- 画像ファイル名が決まっている
- 4コマ、または縦長1枚漫画の大まかな流れが決まっている
- 画像 alt 文の方向性が決まっている

この工程は、記事全体の設計です。画像生成そのものの細かい指示は、後続の「9. 画像生成設計」で行います。

---

### 6. posts/NNN.html草案作成

完了条件:

- `posts/NNN.html` の全文草案を出している
- 既存記事のHTML構造、CSS class、相対パスを踏襲している
- 画像パスが `../assets/images/...` になっている
- title、description、OGP、本文、タグが新記事内容に合っている
- 既存記事本文を勝手に変更していない

この工程では、ChatGPT は `posts/NNN.html` の全文を出力してよいです。

---

### 7. index.html追加カード草案作成

完了条件:

- `index.html` に追加する新記事カードブロックだけを出している
- `index.html` 全文を出していない
- 既存記事カードを削除、短縮、並べ替えしていない
- 貼り付け位置を明示している
- 新記事カードの番号、日付、タイトル、説明、タグ、画像パスが `posts/NNN.html` と一致している

貼り付け位置は、原則として記事一覧セクション内の現在の最新記事カードより上です。

---

### 8. POST_INVENTORY.md追記草案作成

完了条件:

- `docs/agent/POST_INVENTORY.md` に追記する1行、または1ブロックを提示している
- 記事番号、タイトル、公開日、テーマ、今回の距離感、タグを含んでいる
- 似ている既存記事や重複回避メモを含んでいる
- 新記事公開時に棚卸しが更新される状態になっている

`POST_INVENTORY.md` は、新記事追加時の必須成果物として扱います。

---

### 9. 画像生成設計

完了条件:

- 画像生成に使うプロンプトを作っている
- 基準サンプル画像3点を確認対象として明示している
- GitHub連携等で画像内容を直接確認できる場合は、基準サンプル画像の作風、余白、文字量、漫画構成、吹き出しの向きを確認している
- 画像内容を直接確認できない場合は、画像を見た前提で分析せず、人間に画像添付または確認を依頼している
- 日本語文字、縦長SNS向け、かわいいAIキャラクター、軽い自虐、教育感などの条件を含めている
- 吹き出しの文字量が多すぎないよう制御している
- 記事タイトル、本文メモ、今回の距離感と画像内容がズレていない

この工程は、画像生成チャットへ渡すための設計です。記事構成そのものをここで大きく変更しません。

---

### 10. 画像生成・確認

完了条件:

- 人間が画像生成チャットで画像を生成している
- 採用画像を `assets/images/` に保存している
- ファイル名が設計どおりになっている
- 日本語が読めるか、文字切れがないか、既存記事と作風がズレていないかを確認している

画像確認時の主なチェック項目:

- 日本語が読める
- 変な英字ノイズがない
- 吹き出しが切れていない
- 文字量が多すぎない
- テーマが伝わる
- スマホ幅でも読めそう
- 既存漫画PNGと雰囲気が大きくズレていない

---

### 11. 最終HTML調整

完了条件:

- 採用画像に合わせて alt 文を最終調整している
- 画像ファイル名が変わった場合、`posts/NNN.html` と `index.html` の両方に反映している
- title、description、OGP、本文、カード説明、POST_INVENTORY の内容が矛盾していない
- 既存記事や既存カードを不要に変更していない

---

### 12. ローカル確認

完了条件:

- 人間がローカル環境で表示確認している
- トップページに新記事カードが表示されている
- 新記事ページが開ける
- 画像が表示されている
- スマホ幅で読める
- 誤字がない
- 既存記事カードが消えていない
- 既存リンクが壊れていない

確認コマンド例:

```bash
python3 -m http.server 8000
```

確認URL:

```text
http://localhost:8000/
```

---

### 13. commit / push / 公開確認

完了条件:

- `git status` で変更ファイルを確認している
- `posts/NNN.html`、`index.html`、`docs/agent/POST_INVENTORY.md`、画像ファイルが必要に応じて含まれている
- commit メッセージが決まっている
- 人間が `main` に push している
- GitHub Actions / GitHub Pages の反映を確認している
- 公開URLで新記事を確認している

通常の ChatGPT / 人間補助モードでは、commit と push は人間が行います。

---

## 工程順の原則

画像生成は、記事タイトル・本文メモ・今回の距離感・indexカード・POST_INVENTORY追記案が固まってから行います。

理由:

- 画像と本文のズレを防ぐ
- タイトル変更による画像作り直しを減らす
- alt 文、OGP説明、記事カード説明を画像内容と揃えやすくする
- 棚卸し更新漏れを防ぐ

ただし、画像ファイル名は `posts/NNN.html` と `index.html` の草案に必要なため、「5. 記事構成設計」の時点で決めます。

---

## ChatGPT の出力ルール

ChatGPT は、各工程で次に人間が行う作業が分かるように出力します。

特に以下を明示します。

- どのファイルに反映するか
- どこへ貼り付けるか
- 既存ファイルのどこを変更してはいけないか
- 次の工程に進んでよいか
- 人間の確認が必要な点

長文になりそうな場合は、まず現在地と次の1工程だけを示します。

---

## 禁止事項

通常の ChatGPT / 人間補助モードでは、ChatGPT は以下を行いません。

- repository を直接編集する
- 作業ブランチを作成する
- commit する
- push する
- PR を作成する
- merge する
- `index.html` 全文を出力する
- 既存記事カードを削除、短縮、並べ替えする
- 既存記事本文を勝手に改変する

ただし、人間が明示的に GitHub へのファイル作成・更新を依頼した場合は、その依頼範囲に限って対応します。
